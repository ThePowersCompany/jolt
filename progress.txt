2026-05-11 PRD #6 — Static file serving layer (FileServeLayer / FileServeService)

Completed:
- joltr-core/src/file_serve.rs (new) — `FileServeLayer` + `FileServeService<S>`:
  - `FileServeLayer::new(prefix, root)` normalizes the prefix at construction (adds leading `/` if missing, strips trailing `/`s; empty stays empty for the "serve everything from root" mode) and stores the root `PathBuf`.

- URI rewrite (strip prefix) at call time rather than mounting `ServeDir` per-prefix internally. Rationale: `ServeDir` resolves the request path relative to its root and does NOT know about the mount prefix — rewriting once at the layer's `call` boundary keeps `ServeDir` ignorant of routing concerns and lets the same `serve_dir` instance handle every request. Query string is preserved unchanged in case a future PRD (#7 cache headers) wants to read it.
- Inner service swap (`std::mem::replace(&mut self.inner, cloned)`) is unconditional, even on the prefix-match branch where the inner isn't called. Rationale: matches the CorsService discipline so a caller invoking `poll_ready` → `call` → `poll_ready` always operates on a freshly-cloned slot regardless of which branch the previous `call` took. This pins one swap-discipline rule for every layer rather than two.
- `serve_dir.oneshot(req)` (vs. manual `poll_ready` then `call`). Rationale: `oneshot` is the standard tower pattern when calling a service inside an async block where we don't hold a `Context`; it handles readiness polling correctly without us having to fabricate a context. Cheaper than re-implementing the dance, and `ServeDir` with no fallback is always ready so the readiness poll is effectively a no-op.
- `match_prefix` rejects "sibling prefix" hits (path `/staticky` against prefix `/static`) by requiring the suffix to be empty or start with `/`. Rationale: a literal `starts_with` would silently route `/staticky/foo` to `<root>/y/foo`, which is both surprising and a footgun for callers naming routes that share a literal prefix. The explicit segment-boundary check is one extra comparison and pins the spec'd "/static/*" expectation.
- Empty prefix supported as "serve every request from the root." Rationale: zero special API surface — falls out of the `match_prefix` empty-string branch — and lets advanced callers stack `FileServeLayer` as a catch-all without inventing a separate "no prefix" constructor. Not currently exercised in tests beyond the unit-level matcher; users opting into this should be aware their inner service won't receive any requests.
- PRD #7 (cache headers + ETag + Last-Modified) and PRD #8 (gzip + Range) are explicitly out of scope here. `ServeDir` in tower-http 0.6 already emits `Content-Length` / `Content-Type` and supports `Range` and `Accept-Encoding: gzip` opt-ins via `precompressed_gzip()`; #7 will add `Cache-Control`/`ETag` injection (probably as a post-call response header pass), and #8 will toggle the precompression mode + verify range/traversal coverage. No layer-internal pieces from this PRD need to change for either — both are additive on top of the existing dispatch.
- Path traversal protection is delegated to `ServeDir`. Rationale: tower-http's `build_and_validate_path` (mod.rs:455) rejects any `Component::ParentDir`/`RootDir`/`Prefix` segment in the requested path before opening the file. Re-implementing the check in JoltR would be a second source of truth for the same security property; delegating keeps the security boundary in one well-tested place.
